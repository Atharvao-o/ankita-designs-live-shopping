## вңҰ What is AtriumVerse?

AtriumVerse is a **virtual spatial collaboration platform** вҖ” think of it as a persistent 2D office world you actually walk through. You pick a character, spawn into a tile map, and move around freely. **The world drives your communication:**

- Walk **close** to a colleague вҶ’ their voice fades in automatically (proximity audio)
- Step **into a private room** вҶ’ a video conference opens just for that zone
- Send a **message** вҶ’ it's encrypted end-to-end before it ever leaves your device

No meetings to schedule. No calls to join. Just walk up to someone and start talking.

### рҹ”җ &nbsp;End-to-End Encryption
AtriumVerse implements a **full zero-knowledge E2EE system** built entirely on the Web Cryptography API. The server stores ciphertexts it cannot read.

```
X25519 keypair  вҶ’  ECDH shared secret  вҶ’  HKDF(secret, context)  вҶ’  AES-256-GCM
```

| Component | Implementation |
|---|---|
| **Key generation** | `window.crypto.subtle.generateKey` В· X25519 В· non-extractable permanent keys |
| **Key storage** | IndexedDB (browser) via `idb` В· private key never leaves the device |
| **Key backup** | WebAuthn PRF (Face ID / Windows Hello) or PBKDF2 passphrase В· blob stored server-side |
| **Device linking** | ECDH ceremony with ephemeral keypair В· WebAuthn biometric assertion required |
| **Channel keys** | Per-device AES-256-GCM blobs В· per-epoch rotation on member join/leave |
| **DM encryption** | ECDH between sender private key and each recipient device's public key |
| **Forward secrecy** | Channel epoch increments on departure В· old key holders cannot read new messages |
| **Key recovery** | Server stores encrypted blob only В· decryptable by PRF/passphrase locally |

The schema has evolved across **7 Alembic migrations** from initial setup through full E2EE:

```

channel_encryption       per-channel encryption enabled + current epoch
key_backups              encrypted private key blob (PRF or passphrase method)
```

## вңҰ How the E2EE Works (Plain English)

```
1. You open AtriumVerse for the first time.
   вҶ’ Your browser generates an X25519 keypair.
   вҶ’ The PUBLIC key is sent to the server. The PRIVATE key stays in your browser's IndexedDB.
   вҶ’ The server has zero knowledge of your private key.

2. You set up a backup.
   вҶ’ Option A: WebAuthn PRF вҖ” your face/fingerprint produces a deterministic key
               that encrypts your private key. The encrypted blob is stored on the server.
   вҶ’ Option B: Passphrase вҖ” PBKDF2 derives a key. Same idea.

3. You open AtriumVerse on a second device.
   вҶ’ Your new device generates its own keypair.
   вҶ’ It asks your first device for approval.
   вҶ’ Your first device does a WebAuthn biometric check (CANNOT be skipped).
   вҶ’ If passed: ECDH between device 1's private key and device 2's ephemeral public key
               produces a shared secret. Your private key is wrapped in AES-GCM and
               transmitted. Device 2 unwraps it. Both devices now share the same identity.

4. You send a DM.
   вҶ’ ECDH between your private key and each of the recipient's device public keys.
   вҶ’ HKDF(shared_secret, message_id, "dm-epoch:N") вҶ’ per-device AES-256-GCM key.
   вҶ’ Each device gets its own ciphertext slice stored in the database.
   вҶ’ The server stores {user_id, encrypted_ciphertext} and nothing else.

5. You join an encrypted channel.
   вҶ’ The channel has a symmetric "channel key" wrapped with ECDH for every member device.
   вҶ’ When a member leaves, the server owner rotates the epoch. New key. Old member locked out.
   вҶ’ Old messages still readable by old member (their old epoch key is intact).
   вҶ’ New messages: new key, new epoch. Forward secrecy.
```

The WebSocket gateway at `/ws/{server_id}?token=<jwt>` handles all real-time events:

## вңҰ Map Authoring

Maps are created in [Tiled Map Editor](https://www.mapeditor.org/) and exported as JSON.

The backend auto-parses your map on server creation:

```
Map layer: "Zones"
  Objects named "Spawn_*"      вҶ’ spawn points (random selection on join)
  Objects named "*room*"       вҶ’ PRIVATE zones (trigger video conference)
  Objects named "*Room*"       вҶ’ PRIVATE zones
  All other objects            вҶ’ PUBLIC zones (proximity chat only)

Map layer: "Collision"
  Any tile with index > 0      вҶ’ impassable (Grid Engine collision)
```

To add a new map:
1. Create `frontend/public/phaser_assets/maps/your_map.json`
2. Add a thumbnail to `frontend/public/phaser_assets/map_thumbnails/`
3. Register in `frontend/lib/map-config.ts`

## вңҰ Security Model

| Threat | Mitigation |
|---|---|
| Server reads messages | Zero-knowledge E2EE вҖ” ciphertexts only stored, keys never transmitted in plaintext |
| Stolen JWT token | Device approval requires WebAuthn biometric вҖ” JWT alone cannot approve a device |
| TOCTOU race on first device | PostgreSQL partial unique index on `devices(user_id) WHERE is_trusted=true` |
| WebAuthn replay attack | Single-use challenge stored in Redis with TTL, deleted immediately after verification |
| Challenge outliving request | Challenge TTL = `min(60s, seconds_until_request_expiry)` |
| Stale member reading channel | Epoch rotation on every membership change вҖ” removed members have no key for new epoch |
| Device cloning | Authenticator sign count tracked per credential, incremented on each assertion |
| Weak password storage | argon2id with sensible cost parameters |
